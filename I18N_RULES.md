# i18n правила (RU/EN)

## Цель
Приложение должно оставаться двуязычным (русский/английский). Любые новые экраны и изменения UI не должны “ломать” второй язык.

## Обязательное правило
**Любой пользовательский текст** (видимый текст, кнопки, подписи, подсказки, placeholder, title, aria-label, confirm/alert сообщения) **нельзя хардкодить строками** в `.tsx`.

Вместо этого используем:
- `const { t } = useI18n()` и затем `t('some.key')`

## Где хранятся переводы
- `src/i18n/en.ts` — базовый список ключей (типобезопасный)
- `src/i18n/ru.ts` — русские значения для всех ключей из `en.ts`

## Как добавлять новый текст (правильный процесс)
1. Добавь ключ в `src/i18n/en.ts` и английское значение.
2. Добавь перевод этого ключа в `src/i18n/ru.ts`.
3. В компоненте используй `t('ключ')`.

## Автоматическая защита
`npm run lint` запускает проверку `npm run i18n:check`:
- она ищет “хардкод” текст в `.tsx` (JSX-текст и строковые литералы в `aria-label/title/placeholder/alt`, а также `alert/confirm`).
- если найдено — команда завершится с ошибкой.

## Допущения (что можно не переводить)
- Текст в `README.md` и документации.
- Логи `console.*` (не UI).
